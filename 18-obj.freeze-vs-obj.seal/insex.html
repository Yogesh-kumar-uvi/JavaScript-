Understanding Object.freeze() and Object.seal()

What is Object.freeze()?

The Object.freeze() method is used to make an object completely immutable. Once an object is frozen:

  • You cannot add new properties to the object.
  • You cannot modify existing properties of the object.
  • You cannot delete properties from the object.
  • You cannot change property descriptors (like making a property writable again).

In essence, a frozen object is "locked" and cannot be changed in any way.

Use Case:

Use Object.freeze() when you want to ensure that an object remains constant, such as for configuration settings or constants.

What is Object.seal()?

The Object.seal() method restricts changes to an object's structure but still allows modifications to the values of existing properties. Once an object is sealed:

  • You cannot add new properties to the object.
  • You cannot delete existing properties from the object.
  • You can modify the values of existing properties.

In essence, a sealed object has a fixed structure, but its contents can still be updated.

Use Case:

Use Object.seal() when you want to prevent structural changes to an object but still allow updates to its data.

Comparison Between Object.freeze() and Object.seal()

Feature Object.freeze() Object.seal()
Adding New Properties ❌ Not Allowed ❌ Not Allowed
Modifying Existing Properties ❌ Not Allowed ✅ Allowed
Deleting Properties ❌ Not Allowed ❌ Not Allowed
Changing Property Attributes ❌ Not Allowed ❌ Not Allowed

Important Notes

  • Both methods only affect the object itself, not nested objects. To freeze or seal nested objects, you need to use recursion (deep freeze/seal).
  • In strict mode, attempts to modify, add, or delete properties on frozen or sealed objects will throw a TypeError.
